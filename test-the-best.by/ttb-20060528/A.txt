
Условие

Условие:

Легендарный учитель мистер Гаррисон решил проводить дополнительную работу с одаренными детьми в четвертом классе. Гаррисон давно заметил, что Твик - один из самых талантливых ребят Южного Парка. Этот мальчик учителю давно понравился, и поэтому он решил провести с ним занятие по "индивидуальной программе" на тему "Комплексные числа". Но вскоре Гаррисон понял, что практически ничего не знает про эти "магические" числа. Его знания ограничиваются следующим:

    * Комплексным числом z = (x, y) называется упорядоченная пара действительных чисел x и y. Число x называется действительной частью комплексного числа z, а число y - его мнимой частью.
    * Операция умножения комплексных чисел задается по следующему правилу:

      (a, b) * (c, d) = (a * c - b * d, a * d + b * c)

    * Существует естественное взаимно-однозначное соответствие между множеством комплексных чисел и множеством точек на плоскости, на которой введена прямоугольная декартовая система координат. Комплексному числу (x, y) соответствует точка с координатами (x, y).
    * (x, 0) = x, где x - произвольное действительное число, т.е. комплексные числа с нулевой мнимой частью являются действительными. 

Мистер Гаррисон решил дать Твику задание в силу своих знаний. В начале он нарисовал прямоугольник, стороны которого параллельны осям координат, а вершины лежат в точках с целочисленными координатами. Потом он попросил Твика посчитать количество упорядоченных пар комплексных чисел, которые удовлетворяют следующим условиям:

1) произведение чисел из пары является целым числом (т.е. мнимая часть произведения равна нулю);
2) каждое число из пары имеет целую мнимую и целую действительную часть;
3) точки, которым соответствуют числа из пары, лежат внутри или на границе прямоугольника, который нарисовал Мистер Гаррисон.

Когда Твик узнал об этом, он начал сильно нервничать. У мальчика даже начали дергаться плечи и голова. Но папа Твика успокоил сына и сказал ему: "Успокойся, сын! Я попрошу ребят, участвующих в конкурсе TTB, помочь тебе, а ты, лучше, выпей кофе".

Входные данные:

Целые числа x1 и y1 - координаты левого нижнего угла прямоугольника, который нарисовал Мистер Гаррисон.

Целые числа x2 и y2 - координаты правого верхнего угла прямоугольника, который нарисовал Мистер Гаррисон.

Выходные данные:

Количество упорядоченных пар комплексных чисел, удовлетворяющих всем трем условиям, приведенным в условии задачи.

Ограничения:

1. Числа x1, y1, x2, y2 от -1000 до 1000 включительно.
2. Число x1 строго меньше числа x2.
3. Число y1 строго меньше числа y2.
4. Количество тестов не превосходит 25.

Примеры:


1)	x1 = -2
	y1 = -2
	x2 = 0
	y2 = 0
	result = 25

Рассмотрим умножение двух чисел (a, b) * (c, d) = (a * c - b * d, a * d + b * c). Заметим, что a * d ? 0 и b * c ? 0. Чтобы a * d + b * c было равно 0, необходимо, чтобы a * d равнялось 0 и b * c равнялось 0.

Существует 5 способов выбора a и d так, чтобы a * d равнялось нулю:

a = 0, d = 0
a = 0, d = 1
a = 0, d = -1
a = 1, d = 0
a = -1, d = 0

Аналогично, существует 5 способов выбора b и c так, чтобы b * c равнялось нулю. Так как a и d можно выбирать независимо от b и c, получаем 5*5 = 25 способов корректного выбора (a, b) и (c, d).

2)	x1 = 1
	y1 = 2
	x2 = 3
	y3 = 4
	result = 0

В данном случае a * d > 0 и b * c > 0, поэтому a * d + b * c > 0, а значит, ни одного способа корректного выбора (a, b) и (c, d) не существует.

3)	x1 = 0
	y1 = 100
	x2 = 50
	y2 = 150
	result = 2601

В данном случае a * d + b * c = 0 только тогда, когда a = 0 и c = 0 (так как b > 0 и d > 0). Каждую из величин b и d можно выбрать 51 способом, поэтому ответ - это 51 * 51 = 2601.

4)	x1 = 999
	y1 = -1000
	x2 = 1000
	y2 = 999
	result = 4002

5)	x1 = -10
	y1 = -20
	x2 = 30
	y2 = 40
	result = 22881




Решение

using namespace std; class Garrison { public: long long getCountOfPairs (int x1, int y1, int x2, int y2) { long long res; return res; } };
