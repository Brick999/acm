KiaEdit

Installation in TopCoder Arena

  1. Launch arena, log in, choose Options->Editor in the menu.
  2. Click the "Add" button to add a new editor plugin.
  3. Type the following information in the "Enter Plugin Information" dialog:
    Name:KiaEdit
    EntryPoint: kiaedit.KiaEdit
    ClassPath:specify the full path to KiaEdit.jar file, e.g. C:\TopCoder\KiaEdit.jar
  4. Click "OK". Click "Save" in the earlier dialog.
  5. Choose KiaEdit in the list of editors and click "Configure".
  6. Configure plugin for your use, and click "OK" to save changes.

Now, whenever you open a problem during the competition or in practice room, you can launch KiaEdit by choosing "KiaEdit" in the "Choose your editor" list, or by making it a default editor.

Functionality

Like FileEdit (another arena plugin), KiaEdit allows you to use your favorite text editor/IDE with TopCoder Arena. Whenever you open a TopCoder problem, it automatically generates template code for you and saves it in a predetermined file. When you submit or test your code in arena, the code that is being sent to TopCoder's servers is taken from that file.

Automatically generated code includes a testing framework which allows you to locally test your code with examples from the problem statement, as well as your own tests.

KiaEdit currently supports C++, C# and Java.

Testing framework

KiaEdit generates the main() routine and testing framework for you. Just implement the class, specified in the problem statement, and you're ready to compile your code to an executable and test it.

When run without any parameters, your method will be tested on all examples from the problem statement. You can run only a subset of sets by specifying their numbers in the command line. For example "p.exe 0 1 3" will run only example tests 0, 1 and 3. (provided that these tests exist, and p.exe is the name of the executable.)

C++ tester only: If you specify "-" as a parameter, the program will read a test case from the standard input in a format, similar to one, used in problem statement's examples. You can also specify expected output in a Returns:" or "Expected:" clause after the input.

Testing code is inserted between lines "// BEGIN CUT HERE" and "// END CUT HERE". When you submit code to TopCoder's servers, any code between pairs of such lines is removed.

Configuration

Configuration dialog allows you to control the following settings:

Field Description Example
Source file (C++, Java, C#) Path to source file for each language C:\TopCoder\$CLASSNAME$.$EXT$
Template file Path to a file with a template of initial code C:\TopCoder\KiaEdit\template.$EXT$
Footer file Path to a file with KiaEdit's testing code template C:\TopCoder\KiaEdit\footer.$EXT$
Backup file Path to a backup file, or blank if you do not want KiaEdit to make backups C:\Temp\KiaEdit-$CLASSNAME$-$TIME$.$EXT$
Save problem text in HTML to Path to a file where KiaEdit will save problem statement's text in HTML, or blank if you do not wish to save it on your computer C:\Temp\$CLASSNAME$.html
Execute shell commands on opening a problem A sequence of commands (one per line) to be executed when you open a problem. Leave blank if you do not wish to use this feature. firefox "C:\Temp\$CLASSNAME$.html"
gvim "$SOURCEPATH$"

You can specify filenames for template and footer files in the format resource:/path, in this case KiaEdit will look for the specified path in its .jar archive.

Portability

KiaEdit should be able to run on any platform with Java 1.6 and TopCoder Arena. When running on a non-Windows system, use your system's format for file paths in the configuration dialog.

Current version of KiaEdit generates source files with LF line terminators (i.e. Unix style). This may cause problems with certain Windows software - notepad doesn't work, for instance.
